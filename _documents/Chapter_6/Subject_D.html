课题D：安全的无线流量¶

在之前的课题中，你已经为适用于有线和无线架构的网络组件配置了安全性。但是，无线设备和协议带来了它们自己独特的安全挑战。因此，在本章课题中，你将保护基于无线局域网连接的流量。

无线网络¶

从根本上讲，无线网络时指不必单纯凭借网络基础设施中物理布线的网络。相反，无线网络中的数据可以通过不可见电磁频谱中的低频无线电波传输。用这些波发射的信号可以覆盖较小或较大的距离。并且可以有效地穿过墙壁等物体已到达目的地。无线联网的优点包括易于携带，节省昂贵的布线费用，以及破除了物理空间的障碍。这使得无线系统成为手机网络、家庭网络、企业网络和互联网中使用的强大工具。

然而，无线信号的广泛且经常无限制的传播已经导致需要实施专门处理无线网络的新安全策略。

无线天线类型¶

无线天线网信号可以使用各种不同类型的天线进行放大。两种主要天线类别使定向天线和全向天线。定向天线将信号传输到特定点。因此，它们通常具有较大的增益（gain），或者说它们的信号有着可靠的连接范围和以分贝衡量的功率。因此。于全向天线相比，它们不易受干扰。全向天线从各个方向发送和接收无线电波，通常作为无线信号的主要分配源。这些天线在无线路由器和移动无线适配器上很常见，因为这些设备需要在所有可能的方向上提供或接收服务。由于全向天线提供的覆盖范围，它们的增益受到了限制。

天线的分类¶

天线的分类

天线种类

描述

全向

橡皮鸭

橡皮鸭（rubber duck 或 rubber ducky）是一种小型全向天线，通常用橡皮套密封。与全向天线的典型情况一样，橡皮天线增益很小。然后，由于体积小，它们理想地满足了移动性，并常被用于对讲机或其他双向无线电以及短距离无线联网。

全向

圆形天花板

顾名思义，这种全向天线被安装在天花板上，通常用于在建筑物中地房间覆盖无线信号。

定向

八木

主要用于无线点的一种定向天线，在长距离无线联网中也有应用，用于扩大热点的范围。

定向

抛物面

一种非常精确的定向天线，常用于蝶形卫星天线，具有显著的增益。由于它们非常精确，与抛物面天线建立连接要更加困难一些。

定向

背射

一种小型定向天线，看起来与抛物面蝶形天线相似，但增益较少。背射天线用在无线网络中以便能够有效针对特定物理区域，同时不会过度扩大覆盖范围。

定向

易拉罐

这种一种自制的定向天线，它可以扩展无线网络或帮助发现无线网络。顾名思义，易拉罐天线（cantennas）的制作过程通常包括将金属罐放在另一个天线上，例如蝶形卫星天线，以提高其增益。

802.11协议¶

802.11是由电气与电子工程师协会[Institute of Electrical and Electronics Engineers(IEEE)]开发的一种协议族，用于无线设备之间或无线设备与基站之间的无线局域网通讯。在履行安全专业人员的职责中，你可能会在联网实现中遇到各种802.11协议。802.11族中的每种批准协议都有不同的特性，它们被统称为802.11x。下表描述了每种主要协议。

802.11协议¶

802.11协议

描述

802.11a

当时（1999年）批准的一种快速，安全但相对昂贵的无线协议规范。802.11a在5GHz频段上支持高达54Mbps的速度，不幸的是，该速度的范围被限制在仅60英尺内，根据安排接入点的方式，者可能会严重限制用户的移动性。

802.11b

802.11b是第一个被称为Wi-Fi的规范，它是最便宜的无线网络协议。802.11b在2.4GHz频率下提供11Mbps的传输速率。（一些厂商，如D-Link，已将其设备上的速率提高到22Mbps。） 802.11b在开放区域可覆盖1000英尺的范围，在封闭空间可覆盖200至400英尺的范围（墙壁可能会妨碍信号）。它可以向后兼容802.11，但不能与802.11a兼容。

802.11g

在这种规范中，2.4GHz频带内无线数据吞吐量的速率最高可达54Mbps。它与802.11b兼容，且可以以更快的速度运行。

802.11n

这种规范极大地提高了速度，在2.4GHz或5GHz范围内数据吞吐量可达600Mbps。

802.11ac

这种规范通过在5GHz频段中增加更宽地信道来提高802.11n的性能，从而将数据吞吐量提高到1300Mbps。

注解

802.11协议的完整列表，请导航到 https://standards.ieee.org/develop/wg/WG802.11.html 网页中进行查看。

无线加密协议¶

下表介绍了无线联网中常用的加密协议。

无线加密协议¶

协议

描述

有线等效协议（WEP）

使用Rivest Cipher 4(RC4)算法为使用802.11a和802.11b协议的无线通讯提供64位，128位和256位加密。WEP被认为存在安全隐患，因为它依赖初始化向量（IV）来随机化相同的文本字符串，因此已被启用。使用24位IV的WEP在能够预测IV值的IV攻击中非常容易受到影响。

无线保护访问（WPA）

为解决WEP中的一些缺陷而引入的安全协议。WPA是在IEEE 802.11i标准的开发过程中被引入的。提供了密钥的动态重新分配，以防止WEP的密钥攻击漏洞。WPA通过临时密钥完整性协议[Temporal Key Integrity Protocol（TKIP）]提供了改进的数据加密，TKIP是由IEEE 802.11i任务创建的用于替换WEP的安全协议。它与现有的WEP加密相结合，提供了一个128为加密密钥，修复了WEP的密钥长度问题。但是，如果在攻击中，小数据包的内容能被解密，TKIP仍然容易受到影响。

无线保护访问Ⅱ（WPA2）

通过实施802.11i标准的所有强制性组件，在WPA的基础上改进了WPA2。 WPA2增加了高级加密标准（AES）基于密码的计数器模式密码块链消息认证协议[Counter Mode with Cipher Block Chaining Message Authentication Code Protocol(CCMP)]加密，进一步提高安全性并取代TKIP。它提供了一个128位的加密密钥。

注解

虽然802.11i遵循了802.11x的命令惯例，但它处理的更多的是无线安全而非连接方面的问题。

无线认证协议¶

下表介绍了无线网络中常用的认证协议。

无线认证协议¶

协议

描述

可扩展认证协议[Extensible Authentication Protocol（EAP）]

可使客户端和服务器使用一种插件进行互相认证的框架。由于EAP并未指定应当使用的认证方法，因此可以在当前广泛的认证方式中选择，并允许实施将来的认证方式。

IEEE 802.1X

这种标准用于封装基于局域网的EAP通讯。802.1X后来被改编为与无线局域网技术一起使用。802.1X提供基于端口的认证。

受保护的可扩展认证协议[Protected Extensible Authentication Protocol（PEAP）]

由Cisco System、Microsoft和RSA Security组成的联盟开发的开放标准。与802.1X一样，从技术上看，PEAP不是一种EAP方法，而是使用SSL/TLS隧道封装EAP通讯的一种方式。这使PEAP在功能上类似于EAP-TTLS，尽管EAP-TTLS往往能够支持比PEAP更多的认证协议。

RADIUS联合

RADIUS是一种网络认证协议，联合意味着不同网络之间共享信任级别。802.1X标准通常与RADIUS一起使用以执行基于端口的身份验证。

注解

802.1X不能和802.11x混淆，802.11x是指无线联网协议的族。

VPN和开放无线¶

开放无线网络被直接访问时，存在严重的安全风险。由于它们不安全，攻击者可以在网络上任意地发动攻击，并且损害每个用户地通讯。这就是为什么当被迫使使用开放无线网络时，应当尽可能地使用VPN隧道。即使使用不安全地无线热点时，VPN也能提供身份验证技术并加密通过Internet传输的数据。但是，如果VPN不使用安全的隧道协议，就无法提供足够的防御能力。例如，PPTP很容易受到中间人攻击；因此，你应该在使用VPN进行隧道传输时使用更安全的IPSec协议。

无线客户端认证方法¶

在配置WPA或WPA2无线网络时，你还可以选择以下三种客户端认证方法。

  1. WPA/2-个人。由于它依赖预共享密钥[pre-shared key(PSK)]，因此也被称为WPA/2-PSK。这个密钥是你在连接Wi-Fi网络时从你输入的密码生成的。由于每个客户端的密码都是相同的，因此存在攻击者收集密码并获得网络未授权访问的风险。

  2. WPA/2-企业。这要求客户端在被授予访问Wi-Fi网络的权限之前使用802.1X与RADIUS服务器进行身份验证。这有助于缓解密码的猜测或暴力破解攻击。因此WPA/2-企业适用于大型公司网络。

  3. Wi-Fi保护设置（WPS）。这种方法简化了Wi-Fi网络的认证过程。它的工作方式是使客户端输入与特定访问点相关联的PIN码，或按下访问点上的按钮使客户端和接入点建立连接并进行身份验证。正如你所见，WPS上的PIN方法很容易遭受暴力破解，应该避免使用。

注解

所有这三种方式在开放无线网络中都推荐使用，因为开放无线网络不提供任何实质性的认证方式。

无线访问点的安全性¶

下表介绍了一些可用于降低无线网络操作风险的安全方法，特别是有关无线访问点（AP）的安全方法。

无线访问带你的安全性¶

安全方法

描述

MAC过滤

当你在AP上过滤MAC地址时，要么将具有不想要MAC地址的设备列入黑名单，要么将具有授权MAC地址的设备列入白名单。以任何一种方式进行过滤都可以防止未经授权的设备连接到无线网络，即使这些设备知道密码或能够通过其它身份验证方法。MAC过滤可能由几分用处，但由于MAC地址很容易伪装，所以不应该仅依靠这种安全方法。

禁用服务集标识符[service set identifier(SSID)]广播

AP可以广播SSID，SSID通常时人类用户识别一定范围内特定无线网络的方式。这是大多数情况下的默认行为，但你可以选择禁用SSID广播，使用户必须知道并输入SSID才能进行连接。 禁用SSID广播有助于防止随机窥探，但是，与MAC地址过滤一样，这不是一种值得信赖的安全策略。网络扫描器可以轻松揭示试图以这种方式隐藏的无线网络。

信号配置

调整无线信号的强度可以帮助你改变网络的包含范围。例如，超出你的实际场所的信号易受战争驾驶攻击；限制信号的强度从而限制它覆盖的范围，有助于缓解这种攻击。 信号的频带选择和带宽也可能会影响你的无线安全策略。大多数Wi-Fi频带是2.4GHz或5GHz。在现代802.11x实现中，5GHz频率提供比2.4GHz更大的带宽。但2.4GHz频率提供更大的范围，并且可以更容易地穿透墙壁等固件物体。

在胖AP与瘦AP之间做出决定

如果一个访问点将某些任务推卸到另一个设备中，则认为这是个瘦AP。胖AP自己承担这些任务中地大部分。例如，瘦AP可以将认证和加密密钥地生成推卸给集中式服务器，从而使AP简单的管理连接。瘦AP在802.1X认证架构中通常很有用，因为在这种架构中多个AP依靠集中式RADIUS服务器提供身份认证服务。瘦AP由WLAN控制器管理，这是一种配置AP并为无线流量提供交换功能的设备。 尽管瘦AP在基于控制器的无线局域网中可以使用你利用管理式网络架构的优势，但独立存在的胖AP可能有助于降低某些环境中的复杂性。

强制网络门户¶

强制网络门户（captive portal）是一种要求客户端连接到无线网络以便通过网页进行身份验证的技术。除非客户端打开浏览器并完成网页上的必要步骤，否则它们的数据包将被拦截，并且无法正确使用网络。Wi-Fi热点通常会使用强制网络门户，尤其是免费和/或公共的热点，以便在用户开始使用服务之前让用户同意可接受的使用政策。强制网络门户还可以通过要求用户使用正确的证书登录来验证用户身份，防止未经授权的用户加入网络。

站点勘察¶

从一般意义上看，站点勘察（site survey）是指收集某个位置上的信息的过程，包括访问路径，潜在障碍和器材的最佳摆放位置，其目的是尽可能地以最佳方式构建某种东西。无线网络可能会经历一个站点勘察阶段，以便能确切地知道在硬件位置方面如何设计网络。成功的站点勘察为网络及其用户提供了高质量的覆盖范围和带宽，同时还意识到安全协议和要求。通过使用收集RF信号数据的专用工具对建议环境进行建模来完成站点勘察。

例如，无线访问点发出的频率在建筑物中可能会受到墙壁的阻碍，而在开放地带则可以传播地更远。作为一名具有安全意识地专业人员，你可以确定，将访问点放在信号受墙壁限制的地方（超出指定区域）可以防止墙壁另一边的设备使用访问点。在此过程中，你维护了公司的协议，规定不要将无线网络扩展到指定的物理范围之外。

保护无线流量的准则¶

保护无线流量时：

  • 你选择最合适你的基础架构需求的无线天线类型。

  • 选择一种能充分满足你的带宽需要和信号范围要求的802.11x的协议。

  • 使用WPA2加密配置你的Wi-Fi网络。

  • 考虑不同的无线认证协议以及它们如何为你的无线网络提供安全性。

  • 如果你连接到不安全的开放式无线网络，请使用具有强大隧道协议（如IPSec）的VPN。

  • 考虑在大型企业环境中使用WPA2-企业，以便能够利用802.1X/RADIUS认证功能。

  • 避免使用WPS的PIN功能。

  • 不要单纯地依靠MAC过滤，同时还要禁用SSID广播。

  • 选择适当地频带并配置能够满足需求地信号强度。

  • 考虑在基于控制器的架构中使用瘦AP来集中无线网络操作。

  • 实施需要登陆证书的强制网络门户，以防未经授权的用户访问你的Wi-Fi热点。

  • 进行站点勘察，以便能够从安全方面确定布置无线基础设施的最佳方式。
